## Часть B — Кнопки с уже существующим API

### B.1 Каталог — «Добавить товар»

- **Бэкенд:** `POST /catalog/api/v1/catalog/products` (`ProductCreate`) — уже есть.
- **Фронт:** `ApiPaths` + метод в `CatalogApiService.createProduct`, диалог (название, категория, цена, SKU, in_stock), по успеху — `invalidate(catalogProductsProvider)` и опционально переход к карточке.

### B.2 Админка — «Добавить» (пользователь)

- **Бэкенд:** `POST /identity/api/v1/identity/users` (регистрация) — уже есть.
- **Фронт:** диалог (логин, email, пароль) → `IdentityApiService.register` → `invalidate(adminUsersProvider)`; при включённом `AUTH_ENABLED` согласовать с политикой (только роль admin — может потребоваться отдельный эндпоинт в будущем).

### B.3 Финансы — «Инвойс»

- **Бэкенд:** `POST /finance/api/v1/finance/invoices/generate` (`InvoiceRequest`: client_id, order_ids).
- **Фронт:** диалог (номера заказов через запятую или multiline) → вызов API → показать результат / SnackBar; при ошибке — `dioErrorMessage`.

### B.4 Финансы — «Оплата» / «Провести» в модалке

- **Бэкенд:** `POST /finance/api/v1/finance/transactions` (`TransactionCreate`), опционально заголовок `Idempotency-Key`.
- **Фронт:** связать поля в `_showPaymentDialog` с телом запроса, `invalidate(financeSnapshotProvider)` после успеха.
