# Дорожная карта фронтенда GastroRoute

Актуальный чеклист с галочками: **`docs/PLAN_TASKS.md`**.

## Уже реализовано

- Каталог, карточка товара, глобальный поиск в шапке.
- Заказы: список и детали через API.
- Финансы: баланс и транзакции по `client_id` (синхронизируется с `user.id` после входа).
- **Склад:** список остатков `GET /warehouse/api/v1/warehouse`.
- **Логистика:** список маршрутов `GET /logistics/api/v1/logistics`.
- Дашборд: KPI (каталог, заказы, финансы, **рейсы in_progress**); частичные ошибки; pull-to-refresh.
- Ошибки сети: `dioErrorMapper`.
- Вход по флагу `AUTH_ENABLED`, профиль в `AuthState`, редиректы GoRouter.

## Что делать дальше (кратко)

1. **Заказы:** POST создание; при необходимости фильтры на бэке для KPI.
2. **Склад:** POST оприходование из UI.
3. **Админ:** API списка пользователей → экран `/admin`.
4. **Качество:** скелетоны, empty states; JWT на `dio`, когда бэкенд потребует.
5. **Опционально:** persist токенов, лента событий на дашборде.
