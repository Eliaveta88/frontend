# Дорожная карта фронтенда GastroRoute

Актуальный чеклист с галочками: **`docs/PLAN_TASKS.md`**.

## Уже реализовано

- Каталог, карточка товара, глобальный поиск в шапке.
- Заказы: список и детали через API.
- Финансы: баланс и транзакции по `client_id` (синхронизируется с `user.id` после входа).
- Дашборд: KPI из каталога / заказов / финансов; частичные ошибки; pull-to-refresh.
- Ошибки сети: `dioErrorMapper`.
- Вход по флагу `AUTH_ENABLED`, профиль в `AuthState`, редиректы GoRouter.

## Что делать дальше (кратко)

1. **Бэкенд:** публичные REST для **склада** и **логистики** (списки + нужные действия), при необходимости — доработка заказов (POST, фильтры).
2. **Фронт:** подключить новые пути в `ApiPaths`, сервисы, Riverpod, заменить заглушки на экранах Склад / Логистика / Админ.
3. **Качество:** скелетоны, empty states; при готовности бэка — перевод части запросов на `dio` с JWT.
4. **Опционально:** сохранение токенов, лента событий на дашборде.
